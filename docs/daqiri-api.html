API Guide

Complete reference for the DAQIRI Python and C++ APIs. All public classes, methods, parameters, return types, and usage examples are documented here.

Python ≥ 3.10 CUDA ≥ 12.0 C++17 Apache 2.0
Installation

Install via pip (recommended) or build from source. CUDA drivers and a compatible GPU are required.

# Latest stable release
pip install nvidia-daqiri

# With optional extras
pip install "nvidia-daqiri[holoscan,deepstream,rapids]"

# From source
git clone https://github.com/NVIDIA/daqiri && cd daqiri
pip install -e ".[dev]"
💡

For C++ usage, install via the CMake FetchContent or vcpkg package. See the GitHub README for CMake integration instructions.

DAQIRIContext class

The top-level runtime context for DAQIRI. Manages GPU device assignment, plugin loading, and the full lifecycle of all sensor sources registered within a session. Typically one context per process, though multiple isolated contexts on different GPU devices are supported.

class daqiri.DAQIRIContext( gpu_id: int = 0, plugin_paths: list[str] | None = None, enable_metrics: bool = True, log_level: str = "WARNING" )
method register_source (source, name) → SourceHandle ▼

Registers a sensor source with this context. Returns a SourceHandle which can be used to pause, resume, or deregister the source later.

Parameters
NameTypeDescriptionDefault
sourceSensorSourceAn instantiated sensor source or plugin adapter. Must implement the SensorSource interface.required
namestrUnique string identifier for this source within the context. Used to reference the source in bundles and logs.required
Returns

A SourceHandle object bound to the registered source.

method start () → None ▼

Begins streaming from all registered sources. Non-blocking — returns immediately after acquisition threads are launched. Raises RuntimeError if called on an already-running context.

⚠️

Sources must be registered before calling start(). Sources registered after start() must be individually activated via their SourceHandle.activate() method.

method stop (timeout_ms=5000) → None ▼

Gracefully stops all active streams and releases GPU resources. Blocks until all acquisition threads have terminated or the timeout elapses.

Parameters
NameTypeDescriptionDefault
timeout_msintMaximum milliseconds to wait for streams to drain before forcing shutdown.5000
property metrics → MetricsExporter ▼

Read-only property. Returns the MetricsExporter instance attached to this context. Available even before start() is called.

Example
import daqiri

ctx = daqiri.DAQIRIContext(
    gpu_id=0,
    plugin_paths=["/opt/my-sensor-plugins"],
    enable_metrics=True,
    log_level="INFO"
)

cam = daqiri.sources.CameraSource(device="/dev/video0")
handle = ctx.register_source(cam, "front_cam")

ctx.start()
# ... stream from cam ...
ctx.stop()
SensorSource abstract class

Abstract base class for all sensor adapters. Subclass this to integrate custom hardware or software sources. Implement at minimum read_frame() or stream().

class daqiri.SensorSource(ABC):
abstract method read_frame (timeout_ms=1000) → SensorFrame ▼

Reads the next available frame from the sensor. Blocks until a frame arrives or the timeout elapses. Override this method in your subclass.

Parameters
NameTypeDescriptionDefault
timeout_msintMax wait time in milliseconds. Raises TimeoutError on expiry.1000
Raises
ExceptionWhen
TimeoutErrorNo frame received within timeout_ms.
SensorDisconnectedErrorUnderlying hardware was disconnected.
async method stream () → AsyncIterator[SensorFrame] ▼

Returns an async generator that yields frames indefinitely. Internally calls read_frame() in a thread pool executor. Override for a fully async implementation.

Example
async for frame in my_source.stream():
    tensor = frame.gpu_tensor  # CuPy array on GPU
    process(tensor)
property metadata → SourceMetadata ▼

Returns a SourceMetadata dataclass with static information about the source: sensor type, model string, native sample rate, and data shape.

SensorFrame dataclass

Immutable dataclass representing a single acquired sample from a sensor. Carries both the data payload and associated metadata.

Fields
FieldTypeDescription
gpu_tensorcupy.ndarrayThe data payload as a CuPy array residing in GPU memory. Also DLPack-compatible for zero-copy interop with PyTorch, JAX, and TensorRT.
timestamp_nsintHardware or software capture timestamp in nanoseconds since the UNIX epoch.
source_namestrThe registered name of the SensorSource that produced this frame.
frame_idintMonotonically increasing counter local to the source. Resets on source restart.
dtypestrNumPy-compatible dtype string, e.g. "float32", "uint16".
shapetuple[int, ...]Shape of the tensor (H × W × C for image data, N × 4 for point clouds, etc.).
metadatadictSource-specific metadata dictionary. Contents vary by sensor type.
ℹ️

gpu_tensor supports the __dlpack__ protocol. Pass it directly to torch.from_dlpack(), jax.dlpack.from_dlpack(), or TensorRT's Python API without any data copy.

TimestampAligner class

Synchronizes frames from multiple asynchronous sensor sources into aligned bundles using configurable time-windowing strategies.

class daqiri.sync.TimestampAligner( sources: list[SensorSource], window_ns: int = 5_000, strategy: SyncStrategy | str = "nearest", require_all: bool = True, max_queue_depth: int = 32 )
Constructor Parameters
NameTypeDescriptionDefault
sourceslist[SensorSource]List of sources to synchronize. All sources must be registered with the same context.required
window_nsintMaximum timestamp delta (in nanoseconds) allowed between frames in a bundle.5_000
strategySyncStrategy | strAlignment strategy. One of "nearest", "interpolate", or "drop_oldest"."nearest"
require_allboolIf True, only emit bundles when all sources contribute a frame. If False, emit partial bundles with None for missing sources.True
max_queue_depthintPer-source internal buffer depth. Increase for bursty sensors; decrease to reduce memory usage.32
async method stream () → AsyncIterator[FrameBundle] ▼

Yields aligned FrameBundle objects. Each bundle is a dict-like object mapping source names to SensorFrame instances with timestamps within window_ns of each other.

Example
async for bundle in aligner.stream():
    cam_frame   = bundle["camera"]
    lidar_frame = bundle["lidar"]
    delta_ns = cam_frame.timestamp_ns - lidar_frame.timestamp_ns
    fuse(cam_frame.gpu_tensor, lidar_frame.gpu_tensor)
SyncStrategy enum

Defines how the TimestampAligner selects or constructs frames when timestamps from different sources do not exactly coincide.

ValueDescription
NEARESTSelect the frame from each source whose timestamp is closest to the reference timestamp of the bundle. Default strategy.
INTERPOLATELinearly interpolate between adjacent frames from each source to produce a synthetic frame at the exact reference timestamp. GPU-accelerated when tensors are float types.
DROP_OLDESTIf no frame arrives within window_ns, drop the oldest pending frame from each source queue and retry.
TransportMode enum

Specifies the memory transport path used by a source to deliver data to GPU memory.

ValueDescription
CPU_COPYData is acquired to CPU memory and then copied to GPU using cudaMemcpy. Simplest mode; lower throughput.
PINNEDData is acquired into CUDA pinned (page-locked) CPU memory, enabling DMA transfers that bypass the OS paging system.
ZERO_COPYData is written directly into GPU memory via PCIe or NVLink. Requires GPUDirect-compatible hardware.
RDMAData arrives over network (InfiniBand or RoCE) directly into GPU memory using GPUDirect RDMA, bypassing both CPU and host memory entirely.
daqiri::Context C++ class

C++17 equivalent of DAQIRIContext. Factory-constructed via Context::create(). Header: <daqiri/context.hpp>

#include <daqiri/context.hpp> std::shared_ptr<daqiri::Context> ctx = daqiri::Context::create({ .gpu_id = 0, .rdma_enabled = true, .log_level = daqiri::LogLevel::INFO });
static Context::create (const ContextConfig&) → shared_ptr<Context> ▼

Factory method. Returns a shared_ptr to a newly constructed context. Throws daqiri::InitError if the GPU device is unavailable.

method start (FrameCallback callback) → void ▼

Starts all registered sources and invokes callback for each arriving daqiri::Frame on a dedicated thread pool. The callback receives a Frame by value; frame.device_ptr is a raw CUDA device pointer valid for the callback's duration.

Example
ctx->start([](daqiri::Frame f) {
    // f.device_ptr: void* on GPU
    // f.size:       bytes
    // f.timestamp_ns: uint64_t
    my::process(f.device_ptr, f.size);
});
