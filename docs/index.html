Why DAQIRI

Built for Real-Time, High-Bandwidth Workloads

Designed from the ground up to eliminate bottlenecks between physical sensors and GPU compute, DAQIRI provides a unified abstraction layer that scales from embedded edge systems to multi-GPU data centers.

⚡

Zero-Copy GPU Ingestion

Stream sensor data directly into GPU memory using RDMA and GPUDirect, bypassing the CPU for maximum throughput with minimal latency.

🔌

Universal Sensor Abstraction

Unified API for cameras, LiDAR, radar, ultrasound, RF sensors, and custom instruments. Swap hardware backends without changing pipeline code.

🔗

Native NVIDIA SDK Integration

First-class interop with Holoscan, DeepStream, CUDA, TensorRT, and RAPIDS — plug sensor streams directly into your existing NVIDIA pipeline.

📡

Multi-Channel Synchronization

Hardware-level timestamp alignment and software sync primitives to fuse data from multiple asynchronous sensor sources with nanosecond precision.

🧩

Extensible Plugin System

Bring your own sensor drivers via the DAQIRI plugin interface. Hot-swap adapters at runtime to support evolving hardware configurations.

📊

Built-in Observability

Out-of-the-box metrics, tracing, and health monitoring with OpenTelemetry export for production deployments at any scale.

Quick Start

Getting Started

From zero to a running sensor pipeline in under five minutes.

Full Tutorials →
1

Install DAQIRI

Install via pip with CUDA 12+ and Python 3.10+. GPU drivers must be set up beforehand.

pip install nvidia-daqiri
2

Initialize a Context

Create a DAQIRIContext bound to a GPU device. This manages all sensor lifecycles.

import daqiri
ctx = daqiri.DAQIRIContext(gpu_id=0)
3

Register a Sensor Source

Instantiate a built-in adapter or your own plugin, then register it with the context.

from daqiri.sources import CameraSource
cam = CameraSource(device="/dev/video0")
ctx.register_source(cam, "cam0")
4

Stream Frames to GPU

Start the context and consume frames as GPU tensors — zero intermediate copies.

ctx.start()
async for frame in cam.stream():
    # frame.gpu_tensor → CuPy/DLPack
    result = model.infer(frame.gpu_tensor)
Multi-Sensor Fusion Python 
from daqiri.sync import TimestampAligner

aligner = TimestampAligner(
    sources=[lidar, camera, radar],
    window_ns=5_000,
    strategy="nearest"
)

async for bundle in aligner.stream():
    lidar_pts = bundle["lidar"].gpu_tensor
    img       = bundle["camera"].gpu_tensor
    fuse_and_infer(lidar_pts, img)
Holoscan Integration Python 
from holoscan.core import Application
from daqiri.holoscan import DAQIRIOperator

class SensorApp(Application):
    def compose(self):
        src   = DAQIRIOperator(self, source=cam)
        infer = TRTInferOp(self, model="model.trt")
        viz   = HoloVizOp(self)
        self.add_flow(src, infer)
        self.add_flow(infer, viz)
C++ Zero-Copy (RDMA) C++17 
auto ctx = daqiri::Context::create({
    .gpu_id = 0, .rdma_enabled = true
});
auto src = std::make_shared<
    daqiri::RDMASource>(iface);
ctx->register_source(src, "rdma0");
ctx->start([](daqiri::Frame f) {
    process(f.device_ptr, f.size);
});

Custom Sensor Plugin Python 
from daqiri import SensorSource, SensorFrame

class MySensor(SensorSource):
    def __init__(self, cfg):
        self.hw = MyHardwareDriver(cfg)

    async def stream(self):
        while True:
            raw = await self.hw.read()
            yield SensorFrame(
                data=raw,
                timestamp_ns=now_ns(),
                dtype="float32"
            )
